Liberty Corporate Single Service Platform

Client: Liberty Corporate | Role: Product Analyst

An enterprise initiative to unify customer experience and drive digital adoption. This project aimed to enhance customer experience for members, employers, and financial advisors through a unified web app. I conducted requirements analysis, UX design, delivered mockups, and aligned stakeholders to improve usability.

Outcome:

Improved designs and usability and achieved cost savings through negotiated service provider deals.

Feature: View Retirement Summary Dashboard

User Story:
As a member with a group retirement policy,
I want to log in to my portal and view a summary of my retirement goals,
So that I can easily understand my retirement status and make informed decisions to improve my retirement planning.
Scenario: Members with permissions can view retirement summary

Given the user has the required permissions, when the user logs in to the system, then the user should be able to view a comprehensive summary including:

  • Retirement Date
  • Years to Retirement
  • Monthly Contribution Salary
  • Total Projected Benefits
  • Net Replacement Ratio Score
  • Estimated Monthly Income at Retirement
  • Tips to Improve Based on Score
  • Download Benefit Statement

Capitec Bank - Interstellar (Legacy System Re-engineering)

Client: Capitec Bank | Role: Product Owner

I led the re-engineering of a legacy file transmission system for high-volume payment transactions. My responsibilities included analyzing workflows, redesigning the UI, managing the backlog, and leading sprints for implementation.

Outcome:

Streamlined payment operations, reduced licensing costs, and delivered a modernized, user-friendly system.

Feature: View Payments Dashboard

User Story:
As a staff member with the necessary permissions,
I want to view payment files sent and received for the day,
So that I can monitor and manage payment transactions efficiently.
Scenario: View and filter payment files on dashboard

The dashboard displays payment files in a comprehensive table with columns for Date, Direction, Service, Sub-service, State, Start of day, User, ACKs, Nacks, Failed, and Total. Users can filter by Service name and Direction, sort by any column, and navigate through paginated results (10 rows per page).

Key Capabilities:
  • Advanced filtering and sorting across all data columns
  • Pagination for efficient data management
  • Real-time monitoring of payment transaction status
  • Improved UI delivering significant annual savings in licensing and maintenance costs

Capitec Bank - Rapid Payments (PayShap)

Client: Capitec Bank | Role: Product Analyst

This proof of concept project was for South Africa's new immediate payment channel under BankServAfrica. I represented Capitec at industry design sessions, documented requirements, and delivered mockups and integration documentation.

Outcome:

Led POC showcasing instant payment capabilities, Delivered mockups, service integration docs and process flows.

Feature: Make A Payment

User Story:
As a user,
I want to be able to make payments,
So that I can instantly transfer money to beneficiaries once I have successfully logged in, have set up my alias, my account is in good standing and I have sufficient funds available.
Scenario: User makes a payment for the first time

When the user selects the option to add an alias, they can capture their alias, cellphone number, max payment amount, and default reference. The system validates field inputs and provides options to save or clear data with appropriate confirmation messaging.

Scenario: User makes a payment to a beneficiary

Given the user's account is in good standing with sufficient funds, they can select a recipient with prepopulated cellphone number, enter payment amount, edit the default reference, and confirm payment. The system processes the instant payment transaction seamlessly.
